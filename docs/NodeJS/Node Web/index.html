nodejs 介绍#

node 在服务端对系统具备完整的控制能力。

js 可以帮助我们控制浏览器，node 帮助我们控制整个计算机。

表现为：node 可以作为服务器响应前端请求，可以操作数据库或写入文件，还可以使用 RPC （远程过程调用）来使用其他机器上的函数方法等等，只要是计算机可以做到的事，那么 node 以及 node 相关库就可以帮我们做到。

为什么使用 node？

  • 前端友好，学习成本相对较低
  • 支持 BFF 层搭建，FE 可以直接使用 JS 来开发自己想要的服务，可以节省沟通成本，也可以更专注于选择最适合页面展示的服务数据封装来带来更好的优化方案

快速入门#

  1. 写一个 http 服务，最简单的方式是创建一个 http 服务器监听端口并进行响应：
const http = require("http");
http  .createServer((req, res) => {    res.writeHead(200, { "Content-Type": "text/plain" });    res.end("hello world");  })  .listen(8000)  .on("listening", () => {    console.log("server running at http://localhost:8000");  });

  1. 为了封装业务处理逻辑（controller）、复用业务处理逻辑（service）、创建路由（router），koa 帮我们进行了统一的封装，使我们的逻辑解耦变得更易维护，框架做的事情就是帮我们便利的复用行为模式、代码，并使我们的项目可维护性更高

后端模型#

后端的数据处理流程：

前端请求接口发起 request，先进行中间件的过滤处理（参数校验、权限检验等），然后交由后端路由（router）分发到各个具体的业务处理器（controller）上，controller 调用相关服务（service），service 写数据库或者远程调用，最后在经历一系列的中间件过滤将 response 返回给前端。

Router 层是将请求映射到 Controller 层的桥梁，Controller 层是处理 HTTP 请求的入口，Service 层是实现具体业务逻辑的核心。

  • Router 层：比如包含 /posts 和 /posts/:id 两个路由，用于将 HTTP 请求映射到对应的 Controller 层方法进行处理，即在路由中调用相应 Controller 层的方法处理请求，最后返回响应结果；
  • Controller 层：用于处理 HTTP 请求并调用相应的 Service 方法进行业务逻辑处理；
  • Service 层：用于实现具体的业务逻辑；

比如：

const Koa = require("koa");const Router = require("koa-router");const bodyParser = require("koa-bodyparser");const { Sequelize, DataTypes } = require("sequelize");
// 创建 Koa 应用实例const app = new Koa();const router = new Router();
// 配置 bodyParser 中间件，用于解析请求体app.use(bodyParser());
// 创建 Sequelize 实例，用于连接 MySQL 数据库const sequelize = new Sequelize("test", "root", "123456", {  host: "localhost",  dialect: "mysql",});
// 定义 Post 模型，表示博客文章const Post = sequelize.define("Post", {  title: {    type: DataTypes.STRING,    allowNull: false,  },  content: {    type: DataTypes.TEXT,    allowNull: false,  },});
// 同步数据库表结构，如果表不存在则会自动创建sequelize.sync();
// 定义路由router.get("/posts", async (ctx) => {  const posts = await Post.findAll();  ctx.body = posts;});
router.post("/posts", async (ctx) => {  const { title, content } = ctx.request.body;  await Post.create({ title, content });  ctx.body = { message: "Post created" };});
// 注册路由中间件app.use(router.routes());
// 启动应用app.listen(4000, () => {  console.log("Server is running on http://localhost:3000");});

使用 curl 命令发送一个 POST 请求来创建一篇博客文章：

curl -X POST -H "Content-Type: application/json" -d '{"title": "Hello", "content": "World"}' http://localhost:4000/posts

再使用 curl 命令发送一个 GET 请求来获取所有博客文章：

curl http://localhost:4000/posts

以上代码是一个简单的 Koa + Sequelize + MySQL 的后端 API 服务，用于演示如何使用 Sequelize 连接 MySQL 数据库，并定义模型、进行数据读写等操作。在实际项目中，我们还需要添加错误处理、认证授权、日志记录等功能，以确保服务的可靠性和安全性。

Sequelize 是一个基于 JavaScript 的 ORM（对象关系映射）库，可以支持多种数据库，如 MySQL、PostgreSQL、SQLite、MSSQL 等。它允许你在 Node.js 中操作关系型数据库，提供了许多数据库操作方法，比如创建、更新、删除、查询等。

Sequelize 能够将关系型数据库中的表映射到 JavaScript 对象上，使开发者能够使用面向对象的编程方式进行数据库操作，而不需要编写原生的 SQL 语句。此外，Sequelize 还提供了很多强大的功能，比如事务处理、连接池、数据验证、关联查询等。

在 Node.js 中使用 Sequelize 可以大大提高开发效率，减少代码量，同时也能够有效的避免 SQL 注入等安全问题。
